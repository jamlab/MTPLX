# MTPLX 1.0.0

MTPLX v1 is the first full release: a native macOS app and the `mtplx`
command line working as one product, built for Apple Silicon.

## Highlights

- **Native macOS app.** Guided onboarding picks the right model for your
  Mac, sets everything up (engine, fan control, terminal command), and
  tunes decoding for your hardware. No Homebrew required — the app ships
  its own runtime.
- **Fast local inference.** Multi-token prediction decoding with the
  official Qwen 3.5/3.6 and Gemma 4 catalog, a live speed dashboard, and
  an OpenAI-compatible server for your own tools.
- **One product, two surfaces.** `mtplx start` sees the app's running
  server and attaches to it instead of loading a second copy; the app and
  CLI share the same model recommendations and settings.
- **Agents out of the box.** One-click launches for OpenCode, Pi, Hermes,
  and Swival, plus `mtplx bench aime` to score models with fully
  disclosed, coaching-free prompts.

## New commands

- `mtplx stop` — stop the running server cleanly.
- `mtplx settings get/set` — read or change live server settings.
- `mtplx bench aime [--quick]` — run the AIME benchmark from the terminal.

## Server improvements

- OpenAI `stop` sequences are honored across chat, completions, and the
  Anthropic-compatible endpoint.
- `/v1/completions` streams tokens as they are generated, with real
  finish reasons and usage.

## Quality of life

- Busy ports resolve themselves: the app falls back to the next free port
  with a clear banner; the CLI tells you exactly who owns a port and how
  to stop it.
- Automatic app updates via Sparkle, with the runtime refreshed after
  every update.
